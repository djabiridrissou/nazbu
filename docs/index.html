the problem

When the internet drops, your app shouldn’t.

One weak router upstream and the whole operation goes dark — tills, scanners, records, all frozen until the bar comes back. Nazbu makes that impossible.

01The link dies, the business stops. A shop, a warehouse, a clinic, a field team on a shaky connection can’t sell, scan, or record. The app becomes a brick.
02Going offline usually means rebuilding everything. Most solutions ask you to move your data somewhere new and rethink how your app works. Nobody has time for that.
03Your existing systems get left behind. The app that already runs the operation has no simple path to keeping working offline.

how it works

Three moves. Zero changes to your app.

01 · CAPTURE

It reads your data

Nazbu watches the database your app already uses and picks up every change as it happens. Your app keeps saving exactly as before.

nothing to rewrite
02 · SYNC

Devices find each other

Every device carries its own full copy and shares changes directly — over the local network, or across the internet between different networks. No server in the middle, all encrypted.

works across networks
03 · MERGE

Everything converges

When two devices reconnect they catch up automatically, with safe rules for money and records so nothing is lost or double-counted. Files travel with the data that points to them.

safe by default

quick start

This is the whole integration.

Your existing database connection, a room to sync in, and you’re live — offline, on every device, converging on its own.

app.ts
import Nazbu from 'nazbu'
import { Pool } from 'pg'

const room = new Nazbu({
  db: new Pool({ connectionString: process.env.DATABASE_URL }),
  room: 'shop-42',
  internet: true,
  policies: { sales: 'append-only', '*': 'last-writer-wins' }
})

await room.start()   // that database is now offline-first, on every node

No new database, no server to run. Fully typed — TypeScript definitions ship in the box. Turn Nazbu off and your app still works; it just stops surviving the outage.
